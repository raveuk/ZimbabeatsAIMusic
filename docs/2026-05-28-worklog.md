## 2. Applio + Voice Recording attempts

### 2.1 Hardware reality found
- **CPU:** Intel Xeon E5-2697 v2 (2013) — supports **AVX only**, no AVX2, no FMA, no AVX-512. This is the root cause of nearly every issue below.
- **GPU:** RTX 3090 (24 GB) — perfect for RVC training, can't be used because libraries can't load on the CPU.

### 2.2 Applio install + the pedalboard crash
- Installed Applio at `~/Applio`. `./run-applio.sh` died with **"Illegal instruction (core dumped)"**.
- Diagnosed via `PYTHONFAULTHANDLER=1`: crash inside Spotify's `pedalboard` (compiled with AVX2+).
- `pedalboard` is only used for **optional post-processing FX** (Reverb, Chorus, PitchShift, …) on inferred audio. Not needed for training, not needed for plain inference.
- **Patched** both files to import `pedalboard` lazily — only when the FX function actually runs:
  - `~/Applio/rvc/infer/infer.py` (`post_process_audio()`)
  - `~/Applio/rvc/realtime/core.py` (`setup_pedalboard()`)
- After the patch: `app.py imported OK, exit: 0`. UI now launches.
- **Patch saved:**
  - `~/comfy/music-app/patches/applio-no-pedalboard.patch` (canonical, in repo)
  - `~/comfy/music-app/patches/README.md` (re-apply instructions)
  - Also copied to `~/Applio/patches/applio-no-pedalboard.patch`.
- Caveat written into README: **don't use the Realtime voice-changer tab** or enable any post-processing FX in Inference (those still hit AVX2 at runtime).

### 2.3 Mic / audio setup (Linux audio rabbit hole)
A lot of time went into making the onboard ALC662 codec work for recording. Persisted to `~/.config/pulse/default.pa`:

```pa
.include /etc/pulse/default.pa
# Stop Pulse from auto-switching profiles (was dropping the mic source).
unload-module module-switch-on-port-available
unload-module module-suspend-on-idle
# Pin to duplex (mic + speakers).
set-card-profile alsa_card.pci-0000_00_1b.0 output:analog-stereo+input:analog-stereo
set-default-source alsa_input.pci-0000_00_1b.0.analog-stereo
set-default-sink   alsa_output.pci-0000_00_1b.0.analog-stereo
```

Other fixes:
- ALSA `Rear Mic Boost` set to 1 (+10 dB) — at +30 dB the codec hissed badly.
- ALSA `Headphone` channel was muted, unmuted it.
- Default sink had been pointing at HDMI (silent monitor).
- Audacity host should be **ALSA**, device `default` or `pulse` — PulseAudio isn't compiled into Ubuntu's Audacity as a host.

## 4. Auto-start (systemd user services)

The app no longer needs you to babysit `npx expo start` or `npm run dev`. Both auto-start at boot and restart on crash.

### 4.1 New service files
- `~/.config/systemd/user/music-backend.service` — `npm run dev` in `server/`.
- `~/.config/systemd/user/music-expo.service` — `npx expo start --non-interactive` with `EXPO_PACKAGER_PROXY_URL=https://expo.zimbabeats.com`.

Both use the NVM Node at `/home/raveuk/.nvm/versions/node/v26.0.0/bin/`, with that path baked into `Environment=PATH=...`. `Restart=on-failure`, `WantedBy=default.target`.

### 4.2 Pre-existing pieces that already auto-start
- `cloudflared.service` (system) — already running, already enabled.
- User lingering — already `Linger=yes` for user `raveuk` (no `loginctl enable-linger` needed).

### 4.3 Useful commands
```bash
# health
systemctl --user status music-backend
systemctl --user status music-expo

# logs
journalctl --user -u music-backend -f
journalctl --user -u music-expo -f

# restart (e.g. after .env.local change)
systemctl --user restart music-backend
systemctl --user restart music-expo

# disable autostart entirely (if you ever need to)
systemctl --user disable music-backend music-expo
```

### 4.4 Caveat about `--non-interactive`
- Metro keyboard shortcuts (`r` reload, `j` debugger, `m` menu) don't work — there's no TTY.
- **Hot reload from file edits still works** (Metro watches files).
- If you ever need the interactive controls: `systemctl --user stop music-expo`, run `npx expo start` manually, Ctrl+C, then `systemctl --user start music-expo`.
